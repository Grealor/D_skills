# 🪐 UED 智能设计团队 Skills 资产库 (D_skills)

欢迎来到品牌与营销设计团队的智能化设计原子能力与工作流沉淀基地。本仓库旨在通过“代码化、结构化”的方式，将团队的专家经验转化为未来 AI Agent 可直接调用的结构化 Skill。

---

## 🧭 知识库全景架构

本仓库严格遵循确定性、系统化的视觉与工程逻辑，划分为以下三大核心板块：

- `**/workflows`](./workflows/) · 智能工作流规范**
  - 沉淀经过实战验证的跨工具 AI 协同流（如：ComfyUI 资源批量延展、短视频脚本到直出、高端动效预演流）。
- `**/prompts`](./prompts/) · 黄金 Prompt 资产库**
  - 收录具备高度可控性、确定性视觉语言的结构化 Prompt 模板（如：核心 IP Joy 3D 渲染控制、品牌视觉锤概念具象化）。
- `**/automation`](./automation/) · 自动化与跨端配置**
  - 存放连接设计端与开发端的配置文件及底层代码（如：Figma MCP 链接 Codex / Vibe Coding 相关 json 配置）。

## 👥 团队协作与贡献指南 (How to Contribute)

为了维护知识库的高度纯净与格式一致性，全员严格遵守以下主线工作流：

1. **开辟平行宇宙：** 禁止直接在 `main` 分支修改。每次新增 Skill 请在本地 GitHub Desktop 创建新分支（形如 `add/xxx-skill`）。
2. **结构化沉淀：** 所有编写的 `.md` 文档必须包含 `[输入前置条件][核心原子步骤][异常控制/风险规避]` 及 `[可量化提效指标]`。
3. **发起设计评审：** Push 到云端后，提交 **Pull Request (PR)**，由组长/技术导师进行 Code Review（重点审查格式是否抽象、逻辑是否具备确定性、是否存在版权风险）。
4. **并入主库：** 评审通过后 Merge 入 `main` 分支，正式沉淀为团队集体资产。
